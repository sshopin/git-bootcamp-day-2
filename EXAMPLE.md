## Заголовки

В README обычно один `H1` (название проекта), `H2` — разделы, `H3` — подразделы. Глубже почти никогда не нужно.

```markdown
# H1
## H2
### H3
```

> Между `#` и текстом обязательно пробел: `#Заголовок` Git-платформы как заголовок не распознают.

## Текст: жирный, курсив, код

- `**жирный**` → **жирный**
- `*курсив*` → *курсив*
- `~~зачёркнутый~~` → ~~зачёркнутый~~
- `` `inline-код` `` → `inline-код`

Inline-код используется для имён команд, путей, переменных. Например: запустите `git status`, чтобы увидеть состояние рабочего дерева.

## Блоки кода с подсветкой

После открывающих тройных кавычек указывайте язык — GitHub подсветит синтаксис:

```python
def greet(name: str) -> str:
    return f"Hello, {name}!"

print(greet("Git Bootcamp"))
```

```bash
git clone git@github.com:user/repo.git
cd repo
git log --oneline -n 5
```

```yaml
name: ci
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
```

Если язык неважен — оставляйте без идентификатора или ставьте `text`.

## Списки

Маркированный:

- первый пункт
- второй пункт
  - вложенный пункт (два пробела отступа)
  - ещё вложенный
- третий пункт

Нумерованный (Markdown сам пересчитает номера, можно везде писать `1.`):

1. подготовка
1. запуск
1. проверка результата

## Чек-листы

Чекбоксы — особенно полезны в issues и pull requests, там GitHub делает их интерактивными. В README рендерятся как статичные.

- [x] выполнено
- [ ] в работе
- [ ] не начато

## Ссылки и якоря

Внешние ссылки:

```markdown
[Keep a Changelog](https://keepachangelog.com/ru/1.1.0/)
```

→ [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/)

Ссылки на разделы внутри документа (якоря). GitHub берёт текст заголовка, переводит в нижний регистр, заменяет пробелы на `-`, выкидывает пунктуацию:

- заголовок `## Блоки кода с подсветкой` → якорь `#блоки-кода-с-подсветкой`
- заголовок `## Чек-листы` → якорь `#чек-листы`

Пример живой ссылки внутри документа:

```markdown
[Перейти к чек-листам](#чек-листы)
```

→ [Перейти к чек-листам](#чек-листы)

## Цитаты и предупреждения

Блок с `>` в начале строки часто используется как «обратите внимание»:

> Это цитата. Может быть многострочной.
> **Важно:** цитаты хорошо подходят для выделения предупреждений в README.

## Сворачиваемые блоки

Полезно, когда основной README хочется оставить коротким, а длинные подробности (команды установки, настройка окружения, FAQ) спрятать «под спойлер»:

<details>
<summary>Показать пример настройки окружения</summary>

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pre-commit install
```

Внутри `<details>` работает любой Markdown: списки, заголовки, код.

</details>

## Горизонтальная линия

Тремя дефисами на отдельной строке — визуально разделяет крупные блоки:

```markdown
---
```
